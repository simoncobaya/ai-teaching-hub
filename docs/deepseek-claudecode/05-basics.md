## 🚀 启动 Claude Code

### Step 1：创建一个项目文件夹

**macOS / Linux（终端）：**

```bash
mkdir my-first-project
cd my-first-project
```

**Windows（PowerShell 或 CMD）：**

```powershell
mkdir my-first-project
cd my-first-project
```

> 💡 这几个系统的命令其实是一样的！

### Step 2：启动 Claude Code

```bash
claude
```

你会看到 Claude Code 的欢迎界面！🎉

> ⚠️ **重要！两种打字的地方，别搞混了：**
>
> | 在哪里 | 长什么样 | 打什么 | 例子 |
> |--------|---------|--------|------|
> | **终端**（黑色窗口） | 光标前有 `$` 或文件夹名 | 系统命令（无 `/`） | `claude`、`mkdir`、`cd` |
> | **Claude Code 里面** | 光标前有 `>` | `/` 命令 或 中文对话 | `/model`、`/config`、`帮我写代码` |
>
> 简单记：**有 `/` 的是在 Claude Code 里打的，没 `/` 的是在终端里打的。**

### Step 3：信任项目文件夹（首次使用时）

```
Accessing workspace: /你的项目文件夹路径

Quick safety check: Is this a project you created or one you trust?

❯ 1. Yes, I trust this folder
   2. No, exit
```

自己创建的文件夹 → 选 **1. Yes, I trust this folder**（回车）。

> 💡 这是 Claude Code 的安全保护，下次再打开同一个文件夹就不会问了。

> 💡 **快捷用法**：一行命令搞定：
> ```bash
> claude "帮我写一个 Hello World 程序"
> ```

---

### Step 4：验证配置是否成功

**检查模型：**

```
/model
```

如果配置成功，可以看到 sonnet、opus、haiku 等选项（配置了 DeepSeek 后，实际调用的就是 DeepSeek）。

**测试消息：**

```
你好！请用一句话介绍你自己，并告诉我你是哪个模型。
```

- ✅ DeepSeek 回复了 → **配置成功！** 🎉
- ❌ 报错了 → [常见问题解决](./07-troubleshooting.md)

**测试编程能力：**

```
帮我写一个 Python 函数，输入两个数字，返回它们的和。
```

> 💡 **函数**就像一个"小机器"——放原料进去，处理完吐结果。比如榨汁机：苹果进去 → 苹果汁出来。给它两个数字 → 返回它们的和。

如果 DeepSeek 成功写出了代码，一切正常！

> 🎉 恭喜！你现在拥有了一个便宜、聪明、中文好的 AI 编程助手。DeepSeek ↔ Claude 之间随时可用 CC Switch 切换。

---

### 退出 Claude Code

对话结束后，输入 `/exit` 或按 `Ctrl + C` 两次即可退出。

> 💡 输入 `/exit` 后如果有后台任务在运行，Claude Code 会退出前台但后台继续——这很正常。

> 验证成功了！别急着学理论——马上动手做第一个项目！

### 🎯 目标

用 Claude Code + DeepSeek，从零开始做一个 **猜数字游戏**！

完成后你将拥有：
- ✅ 一个可以跑的 Python 小游戏
- ✅ 第一次体验"用 AI 写程序"的全过程
- ✅ 信心！你已经可以用 AI 来编程了！

---

### Step 1：创建项目文件夹

打开一个**新的终端窗口**，创建新文件夹：

```bash
mkdir guess-game
cd guess-game
```

> 💡 为什么要新开终端？因为这个项目需要干净的新文件夹。Windows：`Win` 键输入 `cmd`，Mac：`Cmd+空格` 输入 `终端`，Linux：`Ctrl+Alt+T`。

### Step 2：启动 Claude Code

```bash
claude
```

第一次在这个文件夹启动，会问你是否信任。选 **"Yes, I trust this folder"**。

### Step 3：告诉 AI 你想做什么

```
帮我用 Python 写一个猜数字游戏：

1. 电脑随机选一个 1 到 100 之间的数字
2. 玩家输入猜测的数字
3. 如果猜大了，提示"太大了！"
4. 如果猜小了，提示"太小了！"
5. 如果猜对了，显示"恭喜你猜对了！"和猜了多少次
6. 最多猜 7 次，用完还没猜到就显示正确答案

请写一个完整的、可以直接运行的程序。
```

### Step 4：确认 AI 的操作

Claude Code 会提示你想做什么，比如 `📝 创建文件：guess_game.py`。按 `y` 确认。

- `y` — 确认这次操作
- `n` — 拒绝这次操作
- `a` — 允许之后所有操作（信任 AI 后续操作时用）

### Step 5：看看 AI 写了什么

```
帮我解释一下这个游戏代码是怎么工作的
```

Claude Code 会用简单的话解释代码逻辑。

### Step 6：运行游戏

```
运行这个游戏
```

Claude Code 会帮你执行 `python guess_game.py`。

### Step 7：玩游戏！🎮

```
🎯 我想了一个 1-100 之间的数字，你来猜！
你有 7 次机会。

第 1 次猜测：50
📈 太大了！

第 2 次猜测：25
📉 太小了！

...

第 5 次猜测：34
🎉 恭喜你猜对了！
你用了 5 次猜对了数字 34！
```

## ⚙️ `/config` 设置面板

输入 `/config` 打开交互式设置面板。`↑` `↓`（或 `j` `k`）移动，`←` `→` 切换选项。

### 设置存在哪里？

| 标注 | 存在哪 | 影响范围 |
|:--:|------|------|
| 🌐 用户 | `~/.claude/settings.json` | 你所有项目生效 |
| 📁 本地 | `.claude/settings.local.json` | 仅当前项目、仅你 |
| 💻 会话 | 不存文件 | 关闭即恢复默认 |

> 💡 🌐 = 个人习惯，📁 = 项目特别要求，💻 = 临时调整。

---

### 🌟 最重要的设置

#### 1. 🎓 Output Style（输出风格）⭐

| 项目 | 内容 |
|------|------|
| 生效范围 | 📁 本地 |
| 默认值 | Default |

**先感受一下：** 对 AI 说"帮我写一个计算器"——代码写好了，但你看不懂它为什么这么写。如果能有个老师**边写边解释设计思路**该多好？Output Style 就是用来切换 AI 的"教学模式"的：

| 风格 | AI 的行为 | 适合场景 |
|------|----------|----------|
| **Default** | 标准编程助手 | 日常开发 |
| **Proactive** | 更主动，少问你 | 熟练后 |
| **Explanatory** | 边写代码边解释设计思路 | 🎓 **最适合学习！** |
| **Learning** | 留 `TODO(human)` 让你亲手写 | 有编程基础后 |

> 💡 **强烈建议设为 Explanatory！** AI 像老师一样边做边讲——你不只是"复制代码"，而是真正理解为什么这么写。

来看看区别：

```
Default 模式：
你："帮我写一个计算器"
AI：写好了，这是代码。（没有解释）

Explanatory 模式：
你："帮我写一个计算器"
AI：我先想想计算器需要哪些功能——加、减、乘、除。
    每个功能写成一个函数，这样代码更清晰。
    这里用 if-elif 判断用户选了什么操作……
    （边写边解释为什么这么设计）
    → 你不仅拿到了代码，还理解了设计思路！
```

> ⚠️ **费用提醒**：Explanatory 和 Learning 会让 AI 说更多话，每次聊天大约多花几分钱。DeepSeek 非常便宜——聊一整天就几毛钱。日常简单任务切回 Default，学新知识时用 Explanatory。
>
> 💡 **token 是什么？** token 就是 AI 计字数的单位。1 个中文字 ≈ 2 个 token，1 个英文单词 ≈ 1 个 token。DeepSeek 价格是 100 万个 token 才几毛钱。

> ⚠️ 切换后需要 `/clear` 或重启生效。

#### 2. 🌐 Language（语言）

设为 `Chinese`，Claude 用中文回复你。

#### 3. 🎨 Theme（主题）

换颜色主题。也可以用 `/theme` 命令。

#### 4. 🧠 Model（模型）

选择默认模型层级。也可以用 `/model` 命令切换。

## 🧠 让 AI 记住你

每次打开 Claude Code 它都是"空白的"——完全不记得上次聊过什么。就像朋友每天问你的名字……很烦 😅

两种"记忆"解决这个问题：

---

### 📝 CLAUDE.md — 你写给 AI 的备忘录

**可以放在哪？（不需要记路径，了解一下就行）**

```
① ~/.claude/CLAUDE.md    用户全局：所有项目生效    ← 你的"个人习惯"
② ./CLAUDE.md            项目级：项目根目录        ← 项目"规章制度"
③ ./CLAUDE.local.md      本地：仅你、仅此项目      ← 你的"私人笔记"
```

> 💡 不用管路径！直接对 Claude 说"帮我把XXX加到 CLAUDE.md"，AI 自动处理。

**怎么写？—— 用说的，不用写的！**

```
你："帮我把'代码注释用中文'加到 CLAUDE.md 里"
Claude：好的，已添加 ✅
```

**什么时候加？** 当你说了两三次同样的要求时（比如"注释用中文"、"每次改完先测试"），告诉 Claude 记下来。

Claude 帮你写出来大概长这样：

```markdown
# 我的猜数字游戏

### 项目说明
这是一个猜数字游戏，玩家要在 7 次内猜中 1-100 之间的数字。

### 我的偏好
- 代码注释用中文
- 给代码里的东西起名字用英文（比如叫 score 而不是 分数）
- 每次改完代码，先运行测试再告诉我结果
- 解释代码时用小学生能听懂的话
```

> 💡 你只负责**说**，Claude 负责**写**。查看和管理用 `/memory` 命令。

---

### 🧠 自动记忆 — AI 自己记笔记

当你反复纠正 AI 时，它会自动记：

```
你："不要用 var，用 const"
  ↓ (几次之后)
AI 自动记录：这个用户偏好 const
  ↓ (下次)
AI：直接用 const，不再问你
```

> 💡 CLAUDE.md = 你写的（便签纸 📌），Auto Memory = AI 写的（笔记本 📓）。自动记忆默认开启，`/memory` 查看。

---

### 🎮 动手试一下

如果 AI 做了什么让你不满意（比如注释用了英文），对它说：

```
帮我把"代码注释用中文"加到 CLAUDE.md 里
```

然后输入 `/memory` 看看 Claude 帮你写了什么。

## 🌿 Git 基础 — 项目的"存档"功能

Git 帮你**记录每次改了什么**，随时可以 ⏪ 退回之前版本、🌿 创建分支做实验。

```
没有 Git：改坏了 → 💀 不知道原来什么样
有 Git：  改坏了 → ⏪ 轻松回退
```

Claude Code 自带 git 集成（`/rewind`、`/diff`、`/branch`），而且——**你不需要手动学 Git 命令**，直接用中文告诉 AI：

| 你想做什么 | 对 Claude Code 说 |
|-----------|------------------|
| 初始化 git | "帮我把这个项目用 git 管理起来" |
| 保存改动 | "帮我把现在的改动保存一下" |
| 看改了什么 | "帮我看看改了哪些文件" |
| 退回之前 | "帮我把代码回到上一个保存的版本" |
| 做实验 | "帮我创建一个实验分支来试试新功能" |

---

### 🔍 先检查 Git 装了没

在**终端**里输入（不是 Claude Code 里面）：

```bash
git --version
```

- ✅ 看到 `git version 2.43.0` → 已安装
- ❌ 看到 `command not found` → 对 Claude Code 说 `帮我在电脑上安装 Git`

> 💡 Windows 用户也可直接下载：[https://git-scm.com/download/win](https://git-scm.com/download/win)

### 怎么用？

在新项目里对 Claude Code 说：

```
帮我把这个项目初始化为 git 仓库，再帮我创建一个 .gitignore 文件
```

> 💡 有了 git，改坏了随时回退——就像玩游戏有"存档" 🎮

---

### 🎮 动手试一下

1. 输入 `帮我把这个项目用 git 管理起来`
2. AI 会帮你初始化仓库并创建 `.gitignore`
3. 做一些代码改动后，输入 `帮我把改动保存到 git`
4. AI 会帮你提交——用中文描述改了什么就行

## 🛠️ AI 的"工具箱"

[Claude Code 的起源故事](./03-claude-code.md) 里说 Boris 给了 AI "手和脚"——专业名叫 **Tool Calls（工具调用）**。刚才做猜数字游戏时你已经全部见过了：

| 你看到的 | 背后是 | 作用 |
|---------|--------|------|
| 📝 创建文件 | `Write` | 写入新文件 |
| ✏️ 修改代码 | `Edit` | 精确替换文字 |
| ▶️ 运行命令 | `Bash` | 执行终端命令 |
| 🔍 搜索文件 | `Grep` / `Glob` | 搜索关键词和文件名 |
| 🌐 查资料 | `WebSearch` | 上网搜索 |
| 📖 读网页 | `WebFetch` | 网页内容转文字 |

> 💡 所有工具调用前都会问你确认。还有一个高级功能 **Subagents（子代理）**——AI 能派出几个"小帮手"同时干活，像小组合作。等你做大赛或大项目的时候试试！

## 🔄 上下文管理

Claude Code 就像只有**一张草稿纸**的聪明朋友。草稿纸写满了，前面的内容就擦掉了，AI 就会"忘记"。这张草稿纸的大小叫**上下文窗口**。

用 `/context` 看用量：

| 组成部分 | 大约占用 | 说明 |
|----------|---------|------|
| 系统指令 | ~4,200 tokens | 基础行为规则 |
| 工具定义 | ~15,000 tokens | AI 工具的说明书 |
| CLAUDE.md | ~500-2,000 tokens | 你写的备忘录 |
| 对话历史 | 动态增长 | 聊天记录 |
| 已读文件 | 动态增长 | AI 读过的代码 |

**空间不够了？→ `/compact`** 总结前面的对话，腾出空间。

```
压缩前：[轮1] [轮2] ... [轮20] ← 快满了
压缩后：[轮1~19 摘要] [轮20 全文] [继续聊...] ← 空间回来了！
```

> 💡 像"整理书包"🎒——把不用的整理好，腾出空间装新的。

**完全重来？→ `/clear`** 清空对话，旧对话可在 `/resume` 恢复。

> ⚠️ `/clear` 后 AI 就忘了刚才的事——但 CLAUDE.md 和 Auto Memory 还在，你的偏好不会丢。

| 内容 | `/compact` 后 | `/clear` 后 |
|------|:--:|:--:|
| 项目 CLAUDE.md | ✅ | ✅ |
| Auto Memory | ✅ | ✅ |
| 口头告诉 AI 的事 | ❌ | ❌ |

> 💡 **重要：想让 AI 永远记住的事，写到 CLAUDE.md 里！**

## ❓ 常见问题

### Q：Claude Code 需要联网吗？
**是的！** 需要联网和 AI 模型对话。

### Q：第一次打开会怎样？
- **已配置 DeepSeek**：直接用，不需要 Claude 账号
- **未配置 DeepSeek**：会让你登录 Claude 账号

### Q：为什么有些命令输入了没反应？
用 DeepSeek 时看不到 `/advisor`、`/fast`、`/desktop` 等需要 Claude 订阅的命令——不影响基本功能。

### Q：文件会自动保存吗？
**是的！** 配合 git 使用更安全——随时可回退。

### Q：可以拒绝 AI 的操作吗？
**当然！** 按 `n` 就拒绝。

### Q：`/model` 切换后还是 DeepSeek？
`/model` 切换的是模型层级，实际调用还是 DeepSeek。切换供应商请用 **CC Switch**。

### Q：Output Style 怎么没变化？
切换后需要 **`/clear` 或重启** 才生效——Output Style 在会话开始时加载。

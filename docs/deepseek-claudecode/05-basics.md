## 🚀 启动 Claude Code

### Step 1：创建一个项目文件夹

首先，创建一个文件夹来放你的代码：

**macOS / Linux（终端）：**

```bash
mkdir my-first-project
cd my-first-project
```

**Windows（PowerShell 或 CMD）：**

```powershell
mkdir my-first-project
cd my-first-project
```

> 💡 这几个系统的命令其实是一样的！都可以直接复制粘贴。

### Step 2：启动 Claude Code

```bash
claude
```

你会看到 Claude Code 的欢迎界面！🎉

> ⚠️ **重要！两种打字的地方，别搞混了：**
>
> | 在哪里 | 长什么样 | 打什么 | 例子 |
> |--------|---------|--------|------|
> | **终端**（黑色窗口） | 光标前有 `$` 或文件夹名 | 系统命令（无 `/`） | `claude`、`mkdir`、`cd` |
> | **Claude Code 里面** | 光标前有 `>` | `/` 命令 或 中文对话 | `/model`、`/config`、`帮我写代码` |
>
> 简单记：**有 `/` 的是在 Claude Code 里打的，没 `/` 的是在终端里打的。**

### Step 3：信任项目文件夹（首次使用时）

第一次在一个新文件夹里使用 Claude Code 时，它会问你一个问题：

```
Accessing workspace: /你的项目文件夹路径

Quick safety check: Is this a project you created or one you trust?

❯ 1. Yes, I trust this folder
   2. No, exit
```

- 如果你**自己创建**了这个文件夹 → 选 **1. Yes, I trust this folder**（按回车）
- 如果你**不确定**这个文件夹里有什么 → 选 **2. No, exit**，先看看再说

> 💡 这是 Claude Code 的安全保护，确保它不会在你不知道的文件夹里乱改文件。
> 下次再打开同一个文件夹，就不会再问这个问题了。

> 💡 **快捷用法**：如果你只想让 AI 做一件事，不用进入对话模式，直接写：
> ```bash
> claude "帮我写一个 Hello World 程序"
> ```
> 这样 Claude Code 会直接帮你完成任务，做完就退出！

### Step 4：验证配置是否成功

现在你已经进入了 Claude Code！让我们快速验证一下 DeepSeek 是否正常工作。

**4.1 检查当前模型：**

输入：

```
/model
```

你会看到当前可用的模型层级。如果配置成功，你可以看到 sonnet、opus、haiku 等选项（配置了 DeepSeek 后，这些模型实际调用的就是 DeepSeek）。

**4.2 发送测试消息：**

输入一个简单的问题：

```
你好！请用一句话介绍你自己，并告诉我你是哪个模型。
```

- ✅ 如果 DeepSeek 回复了（通常会提到自己是 DeepSeek），说明 **配置成功**！🎉
- ❌ 如果报错了，请看下一页的 [常见问题解决](./07-troubleshooting.md)

**4.3 测试编程能力：**

```
帮我写一个 Python 函数，输入两个数字，返回它们的和。
```

> 💡 **函数**就像一个"小机器"——你把原料放进去，它处理好之后吐出一个结果。比如榨汁机：放苹果进去 → 出来苹果汁。代码里的函数也是一样：给它两个数字 → 它返回两数之和。

如果 DeepSeek 成功帮你写出了代码，说明一切正常！

## ⌨️ 常用命令

在 Claude Code 里，以 `/` 开头的是特殊命令（slash command）。输入 `/` 就能看到所有可用命令的列表，输入字母可以筛选。

> 💡 **有些命令你可能看不到**。这很正常！可用命令取决于你的平台、计划和环境。比如有些命令需要 Claude 订阅账号才能用（如 `/desktop`、`/teleport`），你用 DeepSeek 就看不到它们。

下面按场景介绍学生最常用的命令 👇

---

### 🟢 会话管理 — 管理你的对话

| 命令 | 作用 | 什么时候用 |
|------|------|-----------|
| `/clear` | 清空对话，重新开始 | 想做一个全新的任务时 |
| `/compact` | 压缩对话，释放空间 | 对话太长、AI 快"忘记"时 |
| `/resume` | 恢复之前的对话 | 退出后想接着之前的话题继续 |
| `/rename 名字` | 给当前对话改名字 | 方便以后从 `/resume` 列表里找到它 |
| `/copy` | 复制上一次 AI 的回复 | 想把 AI 写的代码复制到别处 |
| `/context` | 查看对话空间用量 | 想知道对话还能聊多久 |
| `/rewind` | 回退代码和对话 | 改错了想回到之前的版本 |
| `/diff` | 查看代码改了什么 | 想知道 AI 到底改了哪些文件 |
| `/export` | 导出对话记录 | 想保存这次的对话 |
| `/btw 问题` | 问个临时问题 | 随口问一下，不想影响对话历史 |
| `/recap` | 生成一句话总结 | 离开一会儿，回来看看我刚才做了啥 |

## ⚙️ `/config` 设置面板 — 定制 AI 的行为

输入 `/config` 会打开一个交互式设置面板。用 `↑` `↓` 或 `j` `k` 上下移动，`←` `→` 切换选项。


### 设置存在哪里？

Claude Code 的设置存在三种文件里：

| 标注 | 存在哪 | 影响范围 |
|:--:|------|------|
| 🌐 用户 | `~/.claude/settings.json` | 你所有项目都生效 |
| 📁 本地 | `.claude/settings.local.json` | 仅当前项目、仅你自己 |
| 💻 会话 | 不存文件 | 关闭 Claude Code 就恢复默认 |

> 💡 简单理解：🌐 = 你的"个人习惯"，📁 = 这个项目的"特别要求"，💻 = "临时调整一下"

---

### 🌟 对学生最重要的设置

#### 1. 🎓 Output Style（输出风格）⭐ 最重要！

| 项目 | 内容 |
|------|------|
| 生效范围 | 📁 本地 |
| 默认值 | Default |

**先感受一下：** 你现在对 AI 说"帮我写一个计算器"，AI 很快写好了代码——但你完全看不懂它为什么这么写。如果能有个老师**一边写代码、一边解释设计思路**该多好？

Output Style 就是用来切换 AI 的"教学模式"的。它直接修改 AI 的行为方式，是对 AI 影响最大的设置。有 4 种风格：

| 风格 | AI 的行为 | 适合场景 |
|------|----------|----------|
| **Default** | 标准编程助手 | 日常开发 |
| **Proactive** | 更主动，自己做决定，少问你 | 熟练后使用 |
| **Explanatory** | 边写代码边解释设计思路和原理 | 🎓 **最适合学习！** |
| **Learning** | 留 `TODO(human)` 让你亲手写关键代码 | 有一定编程基础后使用 |

> 💡 **强烈建议设为 Explanatory！** AI 会像老师一样——一边帮你写代码，一边解释"为什么这么做"、"还有哪些选择"。你不需要自己动手写，就能学到编程思路。

> ⚠️ **费用提醒**：Explanatory 和 Learning 模式会让 AI 说更多话（解释、引导），每次聊天大约多花几分钱。DeepSeek 非常便宜——你聊一整天大概也就花几毛钱。如果担心费用，日常简单任务切回 Default，学新知识时再用 Explanatory。
>
> 💡 **token 是什么？** token 就是 AI 计字数的单位。大概 1 个中文字 ≈ 2 个 token，1 个英文单词 ≈ 1 个 token。DeepSeek 的价格是 100 万个 token 才几毛钱——你聊一整天可能也只花几分钱。

来看看 Default 和 Explanatory 的实际区别：

```
Default 模式：
你："帮我写一个计算器"
AI：写好了，这是代码。（没有解释）

Explanatory 模式：
你："帮我写一个计算器"
AI：我先想想计算器需要哪些功能——加法、减法、乘法、除法。
    每个功能写成一个函数，这样代码更清晰。
    这里用 if-elif 来判断用户选了什么操作……
    （一边写代码，一边解释为什么这么设计）
    → 你不仅拿到了代码，还理解了设计思路！
```

> 💡 **Explanatory 模式是学习编程的"加速器"**。就像一个耐心的老师在旁边，边做边讲解——你不只是"复制代码"，而是真正理解 AI 为什么这么写。

> ⚠️ 切换后需要 `/clear` 或重启 Claude Code 才能生效。

## 📂 Claude Code 如何操作文件

Claude Code 会 **自动** 帮你操作文件，但每次操作前都会 **问你确认**：

| 操作 | 你会看到 | 你要做的 |
|------|---------|---------|
| 创建文件 | 📝 "创建文件 xxx" | 按 `y` 确认 |
| 修改文件 | ✏️ "修改 xxx 第 10 行" | 按 `y` 确认 |
| 运行命令 | ▶️ "运行命令 xxx" | 按 `y` 确认 |
| 搜索代码 | 🔍 "搜索 xxx" | 按 `y` 确认 |
| 查网页 | 🌐 "访问网址 xxx" | 按 `y` 确认 |

> 💡 **安全提示**：Claude Code 每次操作都会问你，不会偷偷改你的文件。如果你不确定，可以按 `n` 拒绝。还有几个快捷选项：
> - `a` — 允许之后所有操作（当你信任 AI 接下来的操作时）
> - `n` — 拒绝这次操作
> - `Ctrl+C` — 取消整个任务

## 💡 提示词技巧

和 Claude Code 对话时，怎么说很重要！

### ✅ 好的提示词

| 场景 | 好的提示词 ✅ |
|------|-------------|
| 写代码 | "帮我用 Python 写一个猜数字游戏，范围是 1-100，有 5 次机会" |
| 改代码 | "把第 10 行的颜色从红色改成蓝色" |
| 找 bug | "运行这个程序报错了，错误信息是 TypeError，帮我找问题" |
| 解释 | "用小学生能听懂的话解释这个函数" |
| 做网页 | "帮我做一个九九乘法表网页，要好看一点，用 HTML + JavaScript" |

### ❌ 不太好的提示词

| 场景 | 不太好的提示词 ❌ | 为什么不好 |
|------|-----------------|-----------|
| 写代码 | "写个东西" | 太模糊，AI 不知道写什么 |
| 改代码 | "改一下" | 没说改什么 |
| 找 bug | "有 bug" | 没说是什么 bug |
| 做网页 | "做个好看的" | 没说做什么内容 |

### 💡 好提示词的公式

```
做什么 + 用什么语言 + 具体要求 + 举例说明
```

举个例子：

```
做什么：写一个猜数字游戏
用什么语言：Python
具体要求：范围 1-100，有 5 次机会，猜完告诉答案
```

合在一起就是：

```
帮我用 Python 写一个猜数字游戏：
- 范围是 1-100
- 玩家有 5 次机会
- 猜完要告诉正确答案
```

> 🌱 古人说 **"磨刀不误砍柴工"**。花 10 秒想清楚怎么说，比花 10 分钟改代码划算多了！

### 🔥 进阶技巧：用"角色扮演"让 AI 更懂你

你可以在对话开头告诉 AI "你现在是XX"：

```
你现在是一个小学编程老师，教我学 Python。
请用最最简单的话解释"变量"是什么，再给一个生活中的例子。
```

或者：

```
你现在是一个有 20 年经验的程序员，请帮我审查这段代码，
用严格的眼光找出所有潜在问题。
```

> 💡 这个技巧对大人、小孩都适用！设定角色能让 AI 的回复风格完全不同。

## 🧠 让 AI 记住你

你有没有发现：**每次打开 Claude Code，它都是"空白的"**，完全不记得你上次跟它说过什么？

就像你告诉朋友你的名字，第二天他又问一遍……会很烦 😅

Claude Code 提供了两种"记忆"来解决这个问题：

---

### 📝 CLAUDE.md — 你写给 AI 的"备忘录"

在你的项目文件夹里创建一个 `CLAUDE.md` 文件，写上你想让 AI 记住的事情。

#### CLAUDE.md 可以放在哪？

Claude Code 会在三个地方找 CLAUDE.md，按优先级从低到高：

```
从宽到窄（后面的优先级更高）
┌─────────────────────────────────────────────┐
│ ① ~/.claude/CLAUDE.md    用户全局：你所有项目都生效  │
│ ② ./CLAUDE.md            项目级：放在项目根目录      │
│ ③ ./CLAUDE.local.md      本地：仅你自己、仅此项目    │
└─────────────────────────────────────────────┘
```

> 💡 你不用管这些路径！需要加记忆时，直接在 Claude Code 里说"帮我把XXX加到 CLAUDE.md"，AI 自动帮你找到并编辑对应的文件。

> 💡 用生活类比：
> - ① = 你的"个人习惯"（"我喜欢注释用中文"）
> - ② = 这个项目的"规章制度"（"这个项目用 Flask 框架"）
> - ③ = 你在这个项目里的"私人笔记"（"我还在学 HTML，先别用太复杂的"）

#### 怎么写 CLAUDE.md？—— 用说的，不用写的！

你不需要手动编辑文件！在对话中直接告诉 Claude，它就会帮你写：

```
你："帮我把'代码注释用中文'加到 CLAUDE.md 里"
Claude：好的，已添加到项目 CLAUDE.md ✅
```

**什么时候该加？**

当你发现自己说了**两三次同样的要求**（比如"不要用 var"、"注释用中文"、"每次改完代码先测试"），就告诉 Claude 记下来，下次它就不会忘了。

**Claude 帮你写出来大概长这样：**

```markdown
# 我的猜数字游戏

## 项目说明
这是一个猜数字游戏，玩家要在 7 次内猜中 1-100 之间的数字。

## 我的偏好
- 代码注释用中文
- 给代码里的东西起名字用英文（比如叫 score 而不是 分数）
- 每次改完代码，先运行测试再告诉我结果
- 解释代码时用小学生能听懂的话
```

> 💡 你只负责**说**，Claude 负责**写**。不需要懂 Markdown 格式，不需要会编辑文件。

**怎么查看和修改？**

用 `/memory` 命令打开记忆管理界面——这里能看到你手动加的、Claude 自动记的、以及项目自带的 CLAUDE.md。

Claude Code 每次启动都会先读 CLAUDE.md，就像你写了一张小纸条贴在电脑上 📌

---

### 🧠 自动记忆 — AI 自己记笔记

除了你写的 CLAUDE.md，Claude Code 还有一个 **自动记忆（Auto Memory）** 功能：当你纠正 AI 的时候，AI 会自己记下来。

#### 它怎么工作的？

```
你："不要用 var，用 const"
  ↓ (你说了几次之后)
AI 自动记录：这个用户偏好 const 而不是 var
  ↓ (下次)
AI：直接用 const，不会再问你
```

AI 把这些记忆存在一个专用文件夹里：

```
~/.claude/projects/<项目名>/memory/
├── MEMORY.md           ← 目录，会话启动时加载
├── javascript-style.md ← Claude 写的："用 const 不用 var"
└── build-commands.md   ← Claude 写的："这个项目用 npm run build"
```

> 💡 **CLAUDE.md vs Auto Memory：**
>
> ```
> CLAUDE.md：    你写的 → 告诉 Claude "你要这样做"
> Auto Memory：  Claude 写的 → Claude 告诉自己 "上次我学到了这个"
> ```
>
> 就像便签纸（你写的）和 AI 的笔记本（AI 自己记的）📓

> 💡 自动记忆默认开启，不需要额外配置。用 `/memory` 命令可以查看和管理 AI 写了什么。

## 🌿 Git 基础 — 让你的项目有"时光机"

### 什么是 Git？

Git 是一个**版本控制工具**。简单说，它会帮你**记录每次改了什么**，让你随时可以：
- ⏪ 退回到之前的版本（改错了也不怕）
- 🌿 创建分支做实验（不影响主代码）
- 📝 记录每次改动的说明（"修了一个 bug"、"加了新功能"）

```
没有 Git：  改坏了 → 💀 完蛋了，不知道原来是什么样
有 Git：    改坏了 → ⏪ 轻松回到上一个保存的版本
```

### 为什么学 Claude Code 需要知道 Git？

两个原因：

1. **Claude Code 自带 git 集成** — `/rewind` 回退、`/diff` 看改动、`/branch` 做实验分支
2. **Superpowers 等高级 Plugin 依赖 git** — 如果没有 git，这些强大功能用不了

### 但是！你不需要手动学 Git 命令！

Claude Code 最大的好处就是——**你可以用中文告诉 AI 帮你操作 git**：

| 你想做什么 | 对 Claude Code 说 |
|-----------|------------------|
| 初始化 git | "帮我把这个项目用 git 管理起来" |
| 保存改动 | "帮我把现在的改动保存一下" |
| 看改了什么 | "帮我看看改了哪些文件" |
| 退回之前 | "帮我把代码回到上一个保存的版本" |
| 做实验 | "帮我创建一个实验分支来试试新功能" |

> 💡 AI 会帮你执行 git 命令，你只需要确认（按 `y`）。完全不需要记住 `git add`、`git commit` 这些命令！

### 🔍 先检查：Git 安装了没

在终端里输入（不需要在 Claude Code 里面）：

```bash
git --version
```

如果看到类似 `git version 2.43.0` 的输出 → ✅ 已安装，跳到下一步！

如果看到 `command not found` 或 `'git' 不是内部或外部命令` → ❌ 还没装。在终端里对 Claude Code 说：

```
帮我在电脑上安装 Git
```

Claude 会根据你的系统帮你安装。装好后再回来继续。

> 💡 **Windows 用户**也可以直接下载安装包：[https://git-scm.com/download/win](https://git-scm.com/download/win)（一路点"下一步"就行）

### 怎么让项目支持 Git？

在新项目文件夹里，对 Claude Code 说：

```
帮我把这个项目初始化为 git 仓库，再帮我创建一个 .gitignore 文件
```

AI 会帮你完成所有操作。就这么简单！

> 💡 有了 git，你就能安心让 AI 帮你改代码了——改坏了随时能回退。就像玩游戏时有"存档"功能 🎮

## 🛠️ AI 的"工具箱"

还记得 [Claude Code 的起源故事](./03-claude-code.md) 里说的吗？Boris 给了 AI "手和脚"。

这些"手和脚"的专业名字叫 **Tool Calls（工具调用）**。每次 AI 做这些事情，其实都是在调用一个工具：

| 你看到的 | 背后是 | 工具的作用 |
|---------|--------|----------|
| 📝 创建文件 | `Write` | 写入新文件或完全覆盖旧文件 |
| ✏️ 修改代码 | `Edit` | 精确替换文件中的某段文字 |
| ▶️ 运行命令 | `Bash` | 执行终端命令（运行程序、安装包等） |
| 🔍 搜索文件 | `Grep` / `Glob` | 在代码中搜索关键词 / 按文件名模式查找 |
| 🌐 查资料 | `WebSearch` | 上网搜索最新信息 |
| 📖 读网页 | `WebFetch` | 把网页内容转成文字给 AI 阅读 |

你已经见过这些"工具"了！从第六步的第一个项目开始，AI 就一直用它们帮你编程。了解这个名字，你就更清楚 AI 是怎么工作的了 👏

> 💡 所有工具调用前都会问你确认（除非你设置了自动允许）。你随时可以看到 AI 正在用哪个工具。

## 🔄 上下文管理

### 对话太长怎么办？

Claude Code 就像一个只有**一张草稿纸**的聪明朋友。草稿纸上写着你们聊过的所有内容——每轮对话都在占用草稿纸的空间。草稿纸快写满了，前面的内容就会被擦掉，AI 就会"忘记"。这张草稿纸的大小就叫**上下文窗口（Context Window）**。

用 `/context` 命令查看当前用量：

```
/context
```


| 组成部分 | 大约占用 | 说明 |
|----------|---------|------|
| 系统指令 | ~4,200 tokens | Claude Code 的基础行为规则 |
| 工具定义 | ~15,000 tokens | AI 所有"手和脚"的说明书 |
| CLAUDE.md | ~500-2,000 tokens | 你写的备忘录 |
| 对话历史 | 动态增长 | 你和 AI 的聊天记录 |
| 已读文件 | 动态增长 | AI 读过你的代码 |

### 空间不够了怎么办？→ `/compact`

```
/compact
```

这个命令会让 Claude Code **总结**之前的对话，释放空间继续聊。

```
压缩前：[轮1] [轮2] [轮3] ... [轮20] ← 空间快满了
  ↓ /compact
压缩后：[轮1~19 的 AI 摘要] [轮20 全文] [继续新对话...] ← 空间回来了！
```

> 💡 用"整理书包"来类比 🎒：书本、文具、杂物（对话历史）塞满了书包 → `/compact` 帮你把不用的东西整理好，腾出空间装新的。

### 完全重新开始 → `/clear`

```
/clear
```

这个命令会 **清空** 所有对话，从头开始。旧对话仍然可以在 `/resume` 中恢复。

> ⚠️ `/clear` 后 AI 就完全不记得刚才的对话了，但 CLAUDE.md 和 Auto Memory 还在，所以你的偏好还是会被记住。

### 哪种内容 compact 后还在？

| 内容 | `/compact` 后 | `/clear` 后 |
|------|:--:|:--:|
| 项目 CLAUDE.md | ✅ | ✅ |
| Auto Memory | ✅ | ✅ |
| 对话中口头告诉 AI 的事 | ❌ | ❌ |
| 已读文件内容 | ❌ | ❌ |

> 💡 **重要提醒**：如果你有想让 AI 永远记住的事，**写到 CLAUDE.md 里**！口头告诉 AI 的事——`/clear` 或 `/compact` 后就没了。

## ❓ 常见问题

### Q：Claude Code 需要联网吗？
**是的！** Claude Code 需要联网才能和 AI 模型对话。

### Q：第一次打开 Claude Code 会怎样？

- **已配置 DeepSeek**：直接就能用，不需要登录 Claude 账号
- **未配置 DeepSeek**：Claude Code 会让你登录 Claude 账号（需要 Claude 订阅）。如果你没有订阅，请先去看上一页的配置教程

### Q：为什么有些命令我输入了没反应？

这很正常！Claude Code 的命令因平台和环境而异。用 DeepSeek 时看不到 `/advisor`、`/fast`、`/desktop` 等需要 Claude 订阅的命令——这些不影响基本功能。

### Q：Claude Code 会自动保存吗？
**是的！** Claude Code 创建和修改的文件会直接保存到你的项目文件夹里。配合 git 使用更安全——随时可以回退。

### Q：我可以拒绝 Claude Code 的操作吗？
**当然可以！** 每次操作都会问你，按 `n` 就可以拒绝。

### Q：为什么 `/model` 切换后还是 DeepSeek？

`/model` 切换的是模型**层级**（sonnet/opus/haiku），但这些实际调用的都是 DeepSeek 的模型。要在 DeepSeek 和其他供应商之间切换，请用 **CC Switch**。

### Q：`/config` 里的 Output Style 怎么没变化？

切换 Output Style 后需要 **`/clear` 或重启 Claude Code** 才能生效。这是因为 Output Style 直接修改系统指令，系统指令只在会话开始时加载。

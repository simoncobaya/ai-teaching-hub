# 🔧 管理多个工具

> CC Switch 可以同时管理 7 个 AI 编程工具，一个界面搞定所有配置！

---

## 🎯 目标

学会用 CC Switch 同时管理多个 AI 编程工具。

---

## 🛠️ CC Switch 支持的工具

CC Switch 目前支持以下 **7 个 AI 编程工具**：

| 工具 | 类型 | 说明 |
|------|------|------|
| 🤖 **Claude Code** | 终端工具 | Anthropic 的终端编程助手 |
| 🖥️ **Claude Desktop** | 桌面应用 | Claude 桌面版，支持官方登录和第三方供应商 |
| 📟 **Codex** | 终端工具 | OpenAI 的终端编程工具 |
| 🔍 **Gemini CLI** | 终端工具 | Google 的终端编程工具 |
| 🆓 **OpenCode** | 终端工具 | 开源编程工具 |
| 🦀 **OpenClaw** | 终端工具 | 开源编程助手（多供应商网关） |
| ⚡ **Hermes** | 编程助手 | 支持 MCP、Skills 和 Memory 管理 |

> 💡 CC Switch 会自动检测你电脑上安装了哪些工具。没安装的工具也可以在 CC Switch 中一键安装。

---

## 🔑 核心概念：两种供应商

CC Switch 中有 **两种供应商**：

| 类型 | 说明 | 举例 |
|------|------|------|
| 🌐 **统一供应商** | 跨应用共享的配置，一次添加，多个工具都能用 | 添加一个 DeepSeek 统一供应商，Claude Code、Codex、Gemini 都能用 |
| 🔧 **应用专属供应商** | 仅用于当前选中的应用 | 专为 Codex 配置的 OpenAI 供应商，其他工具看不到 |

> 💡 添加供应商时，面板顶部有两个 Tab 切换：**「应用专属供应商」** 和 **「统一供应商」**。

**统一供应商**就像一张手机卡 📱，可以插到不同的手机里用：

```
添加了一个 DeepSeek 统一供应商
    ↓
勾选要同步的应用：Claude Code ✅  Codex ✅  Gemini ✅
    ↓
修改一次配置，所有关联的应用同步更新！
```

**应用专属供应商**就像专门为某台电脑配的鼠标 🖱️，只能在那个应用里用：

```
为 Codex 单独添加了一个 OpenAI 应用专属供应商
    ↓
只有 Codex 能用，其他工具看不到
```

> 💡 本教程中，我们添加的是 **应用专属供应商**（专门给 Claude Code 用的 DeepSeek）。如果你同时使用多个工具并想统一管理，也可以试试统一供应商！

## 🚀 管理步骤

### Step 1：查看工具状态

打开 CC Switch，在左侧导航栏可以看到所有支持的 **7 个工具**。

每个工具会显示它的当前状态：
- ✅ **已检测到** — 这个工具已经安装在你的电脑上，CC Switch 可以管理它
- ⬜ **未安装** — 这个工具还没有安装

> 💡 对于已安装的工具，CC Switch 可以直接帮你配置供应商和模型。

---

### Step 2：为工具选择供应商

当你添加了供应商（比如 DeepSeek）之后，就可以在不同工具中使用它：

1. 在左侧导航栏，点击你想配置的工具（如 **Claude Code**）
2. 在该工具的页面中，你会看到可用的供应商列表
3. 点击你想使用的供应商的 **「启用」** 按钮
4. CC Switch 会通过本地代理自动将请求路由到该供应商

> 💡 因为供应商是通用的，同一个 DeepSeek 供应商可以同时给 Claude Code、Codex、OpenCode 等多个工具使用。

---

### Step 3：在不同工具之间切换

如果你同时使用多个工具，每个工具可以启用不同的供应商：

| 工具 | 启用的供应商 | 说明 |
|------|------------|------|
| Claude Code | DeepSeek | 日常编程，便宜又好 |
| Codex | OpenAI | 需要用 OpenAI 的功能 |
| OpenCode | DeepSeek | 和 Claude Code 用同一个供应商 |

在 CC Switch 中切换供应商**不需要重启工具**，非常方便！

---

### Step 4：管理工具的安装和更新

CC Switch 还可以帮你管理工具的安装：

- 🔍 **检测安装状态** — 自动发现已安装的工具
- 📥 **一键安装** — 部分工具可以直接在 CC Switch 中安装
- 🔄 **版本更新** — 检测新版本并提醒更新

## 📊 实际使用示例

### 示例 1：全部用 DeepSeek（最省钱）

所有工具都启用 DeepSeek 供应商：

```
Claude Code  → DeepSeek（便宜）
Codex        → DeepSeek（便宜）
OpenCode     → DeepSeek（便宜）
```

因为供应商是通用的，只需要添加一个 DeepSeek 供应商，所有工具都能用！

### 示例 2：不同工具用不同供应商

```
Claude Code  → DeepSeek（日常编程）
Codex        → OpenAI（需要 OpenAI 专属功能）
OpenCode     → 智谱 GLM（中文任务）
```

在 CC Switch 中，分别进入每个工具的页面，启用不同的供应商即可。

### 示例 3：遇到难题时切换

```
日常编程  → DeepSeek + Sonnet（便宜）
遇到难题  → 在 CC Switch 切到 Claude + Opus（最强但贵）
解决之后  → 切回 DeepSeek（省钱）
```

---

## 💡 最佳实践

1. **优先用便宜的供应商** — 日常编程用 DeepSeek，省钱 💰
2. **只在需要时切换** — 遇到特殊情况才用更贵的模型
3. **定期检查用量** — 在 CC Switch 中查看各供应商的余额和用量
4. **保持 CC Switch 更新** — 新版本会支持更多工具和供应商
5. **善用通用供应商** — 添加一次 DeepSeek，多个工具都能用

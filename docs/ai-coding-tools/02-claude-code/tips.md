## 📝 提示词技巧

和 Claude Code 对话时，怎么说很重要！

### ✅ 好的提示词

| 场景 | 好的提示词 ✅ |
|------|-------------|
| 写代码 | "帮我用 Python 写一个猜数字游戏，范围是 1-100，有 5 次机会" |
| 改代码 | "把第 10 行的颜色从红色改成蓝色" |
| 找 bug | "运行这个程序报错了，错误信息是 TypeError，帮我找问题" |
| 解释 | "用小学生能听懂的话解释这个函数" |

### ❌ 不太好的提示词

| 场景 | 不太好的提示词 ❌ | 为什么不好 |
|------|-----------------|-----------|
| 写代码 | "写个东西" | 太模糊，AI 不知道写什么 |
| 改代码 | "改一下" | 没说改什么 |
| 找 bug | "有 bug" | 没说是什么 bug |

### 💡 好提示词的公式

```
做什么 + 用什么语言 + 具体要求 + 举例说明
```

举个例子：

```
做什么：写一个猜数字游戏
用什么语言：Python
具体要求：范围 1-100，有 5 次机会，猜完告诉答案
```

合在一起就是：

```
帮我用 Python 写一个猜数字游戏：
- 范围是 1-100
- 玩家有 5 次机会
- 猜完要告诉正确答案
```

> 🌱 古人说 **"磨刀不误砍柴工"**。花 10 秒想清楚怎么说，比花 10 分钟改代码划算多了！

## 🧠 让 AI 记住你

你有没有发现：**每次打开 Claude Code，它都是"空白的"**，完全不记得你上次跟它说过什么？

Claude Code 提供了两个"记忆"功能来解决这个问题：

### 📝 CLAUDE.md — 你的 AI 备忘录

在你的项目文件夹里创建一个 `CLAUDE.md` 文件，写上你想让 AI 记住的事情：

```markdown
# 我的猜数字游戏

## 项目说明
这是一个猜数字游戏。

## 技术栈
- HTML + CSS + JavaScript
- 用浏览器打开即可运行

## 规则
- 代码注释用中文
- 变量名用英文
- 测试前先运行 npm test
```

Claude Code 每次启动都会先读这个文件，就像你写了一张小纸条贴在电脑上 📌

> 💡 **写什么好？** — 你的项目是做什么的、你喜欢的编程风格、常用的命令。

### 🧠 自动记忆 — AI 自己记笔记

除了你写的 CLAUDE.md，Claude Code 还有一个**自动记忆**功能：当你纠正 AI 的时候，AI 会自己记下来。比如你说了 3 次"不要用 `var`，用 `const`"，AI 会自己记到备忘录里，下次就不会忘了。

## 🛠️ AI 的"工具箱"

还记得 [Claude Code 的诞生故事](./README.md) 里说的吗？Boris 给了 AI "手和脚"。

这些"手和脚"的专业名字叫 **Tool Calls（工具调用）**。每次 AI 做这些事情，其实都是在调用一个工具：

| 你看到的 | 背后是 |
|---------|--------|
| 📝 创建文件 | `Write` 工具 |
| ✏️ 修改代码 | `Edit` 工具 |
| ▶️ 运行命令 | `Bash` 工具 |
| 🔍 搜索文件 | `Grep` / `Glob` 工具 |
| 🌐 查资料 | `WebSearch` 工具 |

你已经见过这些"工具"了！了解这个名字，你就更清楚 AI 是怎么工作的 👏

## 🤖 AI 的小帮手（了解）

Claude Code 还有一个厉害的功能：**Subagents（子代理）**。

当你让 AI 做一件大事时，AI 可以**派出几个"小帮手"** 同时去干不同的活。就像老师布置一个项目，让一些同学查资料、一些同学写代码、一些同学做测试。

> 💡 这个功能比较高级，你现在只需要知道"有这个功能"就行！

## 🔄 上下文管理

### 对话太长怎么办？

当你和 Claude Code 聊了很久，它可能会"忘记"前面的内容。这时候：

```
/compact
```

这个命令会让 Claude Code **总结**之前的对话，释放空间继续聊。

### 完全重新开始

```
/clear
```

这个命令会 **清空** 所有对话，从头开始。

## 🎯 常见场景的提示词模板

### 场景 1：从零开始做项目

```
我想做一个 [项目名称]，用 [编程语言]。

功能要求：
1. [功能 1]
2. [功能 2]
3. [功能 3]

请先帮我设计项目结构，然后逐步实现。
```

### 场景 2：修复 Bug

```
我的程序出了问题：

错误信息是：
[粘贴错误信息]

相关代码在 [文件名] 文件里。

请帮我找到问题并修复。
```

### 场景 3：优化代码

```
请帮我优化 [文件名] 里的代码：
- 让它运行更快
- 让代码更容易阅读
- 加上中文注释
```

### 场景 4：学习理解

```
请用小学生能听懂的话解释 [概念名称]：
1. 它是什么
2. 为什么要用它
3. 用一个生活中的例子来说明
4. 写一段简单的示例代码
```

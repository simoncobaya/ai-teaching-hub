# 🎨 Frontend-Design — 让 AI 做出"不像 AI 做的"界面

> Anthropic 官方 Skill，市场安装量 **94.8 万**。教会 Claude 摆脱"AI 味"的设计。

---

## 🤔 它解决了什么问题？

你有没有注意过——AI 生成的网页界面，看起来都差不多？

```
❌ 没有 frontend-design：奶油色背景 + Inter 字体 + 紫色渐变 + 圆角卡片
✅ 有 frontend-design：  独特配色 + 个性字体 + 大胆布局
```

AI 模型在训练时看过了太多"平均好"的设计，所以它们倾向于生成"安全但平庸"的界面。**Frontend-Design 技能就是来打破这个循环的**——它把 Claude 定位成小型设计工作室的主管，让它做出**有意的、独特的、为这个项目量身定制的**设计选择。

## 🎨 设计原则

以下是 SKILL.md 中 Claude 被"教会"的核心设计理念（中文为原文释义，非逐字翻译）：

### 1. 🌱 从主题出发（Ground it in the subject）

如果任务没有说清楚这个产品是什么，先自己确定——说出一个具体的主题、它的用户、这个页面的唯一任务。主题本身的世界（材料、工具、风格、语言），才是独特设计的来源。

**对学生来说**：就像写作文要先确定"写什么"和"写给谁看"。做一个宠物商店的网页 vs 做一个太空游戏官网，设计应该完全不同。

---

### 2. 🦸 首屏即论点（The hero is a thesis）

打开页面时，用主题世界中最具特色的东西开场——可以是大标题、图片、动画、实时演示、交互。不要不假思索地套用"大数字+小标签+辅助数据+渐变"的模板答案。

**对学生来说**：页面的第一眼决定了别人要不要继续看下去。第一屏放最有代表性的东西。

---

### 3. 🔤 排版承载个性（Typography carries personality）

展示字体和正文字体要有意搭配——不要选那些你会在任何项目里用的字体。设置清晰的字号层级，让排版本身成为设计中被记住的一部分，而不只是内容的"中性容器"。

**对学生来说**：标题用有趣的字体吸引眼球，正文用舒服的字体让人读下去。

---

### 4. 📐 结构本身就是信息（Structure is information）

编号、分隔线、标签——应该编码关于内容的真实信息，而不是装饰。01 / 02 / 03 只在内容真的是一个序列（流程、时间线）时才合适。问自己"这里加编号有意义吗？"

**对学生来说**：不要为了好看加装饰。每个设计元素都应该有理由——问自己"为什么这里要有一条线？"

---

### 5. 🎬 动效要有意地使用（Leverage motion deliberately）

一个精心编排的瞬间比零散的微交互更有冲击力。选一个时刻做到极致——页面加载的入场、滚动触发的揭示、悬停的反馈——而不是到处撒动画。

**对学生来说**：动画是盐，不是饭。放一点点提味，多了反而难吃。

---

### 6. 📏 复杂度匹配愿景（Match complexity to vision）

极繁主义需要精细的执行；极简主义需要间距、字体、细节上的精确。优雅就是把选定的方向执行到位。

**对学生来说**：想做复杂的设计就要做足细节；想做简单干净的设计就要在间距和对齐上下足功夫。

---

### 7. 🎯 一处大胆，其余克制（Restraint and self-critique）

把你的"大胆"花在一个地方。让签名元素成为那一个被记住的东西，周围的一切安静而有纪律。SKILL.md 里引用了 Chanel 的建议：出门前照照镜子，摘掉一件配饰。

**对学生来说**：你的海报可以有一个超级酷的大标题，但其他文字就简简单单。什么都想突出，就等于什么都没突出。

---

### 8. ✍️ 文案也是设计（Writing in design）

界面上的文字只有一个目的：让人更容易理解、更容易使用。它们不是装饰，是设计材料。好文案的特征：主动语态（"保存更改"不是"提交"）、从用户视角出发（"管理通知"不是"Webhook 配置"）、保持一致（"发布"按钮 → "已发布"提示）。错误提示不道歉、不模糊，说清楚发生了什么、怎么解决。

**对学生来说**：按钮上写什么、报错说什么——这些都是设计。把话说明白，用对方能懂的语言。

## 🚫 三种 AI 默认风格

SKILL.md 指出，AI 生成的设计容易不假思索地落入三种风格。它们本身不差——差在变成了"默认"而非"有意的选择"：

| # | 风格 | 特征 |
|---|------|------|
| 1 | 🥛 奶油色 + 赤陶色 | `near #F4F1EA` 暖色背景 + 高对比衬线标题 + 陶土色强调 |
| 2 | 🟢 深黑 + 荧光强调 | 近黑色背景 + 单一荧光绿或朱红色强调 |
| 3 | 📰 宽页式报纸布局 | 细线分隔 + 零圆角 + 密集如报纸的排布 |

> 💡 SKILL.md 原话：这三种在某些项目里是合理的，但它们成了默认而非选择。如果不加思考就用，和人类设计师"被雇来做设计却只重复自己擅长的那一套"没有区别。

## 🔄 设计流程：两遍法

SKILL.md 教 Claude 走一个**两遍流程**：

```
第一遍：头脑风暴 + 计划
  ├── 配色系统（4-6 个命名颜色）
  ├── 排版方案（展示字体 + 正文字体 + 辅助字体）
  ├── 布局概念（用文字描述或 ASCII 线框图对比方案）
  └── 签名元素（这个页面被人记住的那个点）

审查：这个方案独特吗？
  ├── 有没有掉进"三种默认风格"？
  ├── 换一个类似的 prompt 会不会得到差不多的方案？
  └── 如果不够独特 → 修改，说明改了哪里、为什么

第二遍：写代码
  ├── 严格按照设计方案，每个颜色和字体选择都从方案中来
  └── 注意 CSS 选择器特异性，避免选择器互相抵消
```

SKILL.md 还建议：在构建过程中审视自己的作品，如果环境支持就截个图——"一张图值 1000 个 token"。

> 💡 这就像建筑师先画图纸，对照需求检查，再开始施工。而不是一边盖一边想。

## 🎮 适合学生吗？

**这是所有 Skill 里最适合学生的之一。** ⭐⭐⭐⭐⭐

| 原因 | 说明 |
|------|------|
| 🎨 即时可见 | 描述需求 → 看到一个漂亮的网页，成就感满满 |
| 🧠 教思维方式 | 不教"怎么写代码"，教"怎么想设计" |
| 🚫 不需要额外工具 | 纯 HTML/CSS 就能出效果 |
| 🏗️ 零门槛 | 不需要 API Key、数据库、付费账号 |

**推荐的入门玩法：**
1. 对 Claude 说："帮我做一个宠物商店的主页"
2. 观察 Claude 是不是先问你问题（什么宠物？目标用户？风格偏好？）
3. 回答问题，看 Claude 生成的设计
4. 如果不够好，说"换个更活泼的风格试试"

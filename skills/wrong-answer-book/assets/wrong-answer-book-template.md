# 错题库输出模板

> 本模板定义错题库 Markdown 文件的标准结构，格式直接对齐腾讯云 `SubmitQuestionMarkAgentJob` API 的返回结构。  
> **第一优先级：可读性** — 排版清晰、层次分明。  
> **第二优先级：可维护性** — 内容结构固定，便于程序读取和人工校对。

---

## 文件命名规范

```
错题库_{年级}{科目}_{试卷名称}.md
```

示例：`错题库_三年级语文_期中测试题.md`

---

## 文件结构

### 1. 文档头部

```markdown
# 错题库 — {年级}{科目} {试卷名称}

> **录入时间**：{日期时间}
> **试卷**：{年级}{科目} {试卷名称}
> **总题数**：{N} 道 | **错题数**：{M} 道

---
```

**设计说明**：
- 一行标题包含所有关键信息
- 元信息用 blockquote，视觉轻量

---

### 2. 错题统计总览

```markdown
## 错题统计

| 序号 | 大题 | 小题 | 知识点 | 作答 | 状态 |
|:---:|------|------|--------|:---:|:---:|
| 1 | 根据语境拼写词语 | (2) zǎo chén | 汉字拼音拼写 | ❌ | — |
| 2 | 按查字典的知识填空 | 先查大写字母 | 音序查字法 | ❌ | ✅ 已掌握 |

**薄弱知识点分布**：

| 知识点 | 错题数 |
|--------|:---:|
| 汉字拼音拼写 | 1 |
| 音序查字法 | 1 |

---
```

**设计说明**：
- 「大题」「小题」直接对应 API 的 `MarkItemTitle` 层级结构
- 「知识点」来自 API 返回的 `KnowledgePoints`
- ❌ 图标直观标记错题
- 「状态」列：`—` 表示未掌握，`✅ 已掌握` 表示用户已确认掌握
- 薄弱知识点分布帮助快速定位重点

---

### 3. 错题逐题分析

> ⚠️ **可读性原则**：每道错题必须**独立完整**，即使与前一道错题属于同一大题，也要重复写出大题标题和原题内容。禁止「同上」「同大题」等省略写法。

```markdown
### 错题 {序号} — {知识点}

**大题**：{MarkInfos[].MarkItemTitle}

**原题**：
> {完整原题文本，保留 API 返回的原始空位格式（如 ____）}

**答错空**：{MarkInfos[].MarkInfos[].MarkItemTitle，定位具体空位}

| 项目 | 内容 |
|------|------|
| **我的答案** | {HandwriteInfo} |
| **正确答案** | {RightAnswer} |
| **错误原因** | {AnswerAnalysis} |

> 💡 **订正**：{正确思路或口诀}

**知识点标签**：`#{KnowledgePoints[0]}` `#{KnowledgePoints[1]}`

---
```

**设计说明**：
- `---` 分隔每道错题
- 大题/原题/答错空 三级定位，精确到具体空位
- 知识点标签方便检索
- 已掌握的错题标题改为 `### 错题 {序号} — {知识点} ✅ 已掌握`

---

### 4. 薄弱知识点诊断

```markdown
## 薄弱知识点诊断

| 薄弱点 | 严重程度 | 说明 |
|--------|:---:|------|
| 汉字拼音拼写 | 🟡 中 | 1 道，同音形近字混淆 |
| 音序查字法 | 🔴 高 | 2 道，不理解音序=首字母大写 |

---
```

**设计说明**：
- 严重程度：🔴 高（≥2错）/ 🟡 中（1错）
- 说明列给出具体原因，不止数字

---

### 5. 巩固练习建议

```markdown
## 巩固练习建议

### 拼写专项

1. **同音字辨析**：练习「辰/晨」「蜡/腊」等对比组词
2. **看拼音写词语**：每天 5 组

### 音序查字法专项

1. **三要素口诀**：拼音→首字母大写→音节（去声调）
2. **字典实操**：用字典查 5 个字体会步骤

---
```

**设计说明**：
- 按薄弱知识点分组
- 建议具体可操作

---

### 6. 原始试卷图片

```markdown
## 📷 原始试卷

![试卷图片](images/{MD5}.{ext})

> 可对照回顾完整试卷内容。

---
```

**设计说明**：
- 放在文档末尾，复习时对照查看
- 图片路径为相对路径指向 `images/`

---

### 7. 文档尾部

```markdown
*本文件由「智能错题库」自动生成*
```

## 字段与 API 映射

| 输出字段 | API 来源 |
|----------|----------|
| 大题 | `MarkInfos[].MarkItemTitle` |
| 原题 | `MarkInfos[].MarkItemTitle`（完整题干） |
| 答错空 | `MarkInfos[].MarkInfos[].MarkItemTitle` |
| 我的答案 | `AnswerInfos[].HandwriteInfo` |
| 正确答案 | `AnswerInfos[].RightAnswer` |
| 错误原因 | `AnswerInfos[].AnswerAnalysis` |
| 知识点 | `AnswerInfos[].KnowledgePoints` |
| 作答（对/错） | `AnswerInfos[].IsCorrect` |

---

## 可读性设计原则

| 原则 | 实施方式 |
|------|----------|
| **标题层级清晰** | `#` → `##` → `###` 三级 |
| **视觉分隔** | 错题之间用 `---` 分隔 |
| **表格优先** | 统计、诊断用表格 |
| **独立可读** | 每道错题完整包含大题+原题，不依赖上下文 |
| **留白合理** | 各板块间保持空行 |

## 五、选题避坑指南

### 5.1 不建议选择的主题类型

| 类型 | 原因 | 举例 |
|------|------|------|
| **纯娱乐类** | 缺乏教育意义 | "讲笑话机器人"、"聊天机器人" |
| **单功能查询** | 节点数难达标，缺乏智能决策 | "查天气"、"翻译一句话" |
| **简单判断** | 不体现LLM的智能决策 | "判断输入是否大于10" |
| **争议性主题** | 涉及敏感话题 | 政治评论、医疗诊断 |

### 5.2 选题的加分要素

| 要素 | 加分原因 |
|------|----------|
| **解决真实问题** | 有实用价值，创新性维度加分 |
| **多学科融合** | 跨学科，创新性10%加分 |
| **多模态输入** | 图片/语音，先进性5%加分 |
| **多智能体协同** | 多个LLM节点分工，先进性5%加分 |
| **输出校验** | 代码节点验证LLM输出，扩展性5%加分 |
| **智能人机交互** | 语音/图文/多轮对话，先进性5%加分 |
| **软硬结合** | 连接传感器/设备，扩展性5%加分 |
| **有使用记录** | 他人使用过的截图，传播性5%加分 |

### 5.3 选题检查清单

在阶段1"收敛"环节，用以下清单逐一检查候选主题：

- [ ] 是否符合竞赛一票否决要求？（非AI生成、非纯代码、非人文艺术）
- [ ] 是否包含一定复杂度？（预估节点数是否达标）
- [ ] 是否能包含至少1个基于LLM输出的判断节点？
- [ ] 是否有明确的用户场景和实用价值？
- [ ] 学生是否对这个主题有兴趣？
- [ ] 是否涉及2个以上学科？
- [ ] 能否在规定时间内完成？
- [ ] 是否能在后续添加加分项？（多模态、多智能体、输出校验等）

## 七、从灵感主题到具体智能体的转化示例

以"AI营养师"为例：

| 阶段 | 内容 |
|------|------|
| **灵感主题** | AI营养师 |
| **具体化** | 针对初中生的"一日三餐营养评估与建议"智能体 |
| **核心功能** | 输入身高体重+今日饮食 → 分析营养摄入 → 判断是否均衡 → 给出明日食谱建议 |
| **预估节点数** | 10-12（意图识别→数据收集→营养计算代码→LLM分析→条件选择(均衡/不均衡)→食谱建议→知识库检索） |
| **创新亮点** | 个性化营养方案 + 食物知识库 + 动态调整 |
| **涉及学科** | 生物（营养学）、数学（卡路里计算）、信息技术（工作流开发） |
| **加分项覆盖** | 多学科融合✓、输出校验✓、智能人机交互✓、知识库调用✓ |

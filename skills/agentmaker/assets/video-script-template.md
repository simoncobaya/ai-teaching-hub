# 视频脚本模板

> 本文档供 `/agentmaker` Skill 阶段6使用：生成竞赛报告视频脚本  
> 视频时长：5分钟（严格控制在4分半以内，留30秒缓冲）  
> 格式：MP4，分辨率 ≥ 1280×720，大小 < 200MB

---

## 视频结构总览

| 段落 | 时间 | 内容 |
|------|:---:|------|
| 开场 | 0:00-0:30 | 自我介绍 + 作品概览 |
| 需求分析 | 0:30-1:00 | 为什么要做这个智能体 |
| 系统演示 | 1:00-3:00 | 实际演示智能体的运行过程 |
| 技术讲解 | 3:00-4:00 | 工作流设计和技术亮点 |
| 总结反思 | 4:00-4:30 | 收获与展望 |
| **总计** | **4:30** | — |

---

## 详细脚本

### 段落一：开场（30秒）

#### 口播台词

```
大家好，我是来自[学校名称]的[选手姓名]。
今天我带来的作品是「[智能体名称]」，
它是一个[一句话功能描述]的AI智能体。

[1句话说明核心价值，如：它能帮助XXX解决XXX问题]

接下来，我将为大家展示它的设计和实现过程。
```

#### 画面内容

- [ ] 选手正面出镜（或屏幕录制 + 画中画头像）
- [ ] 展示项目名称和概述文字
- [ ] 显示学校名称

> [镜头提示] 眼神看镜头，语速适中，面带微笑

#### 字幕内容

```
作品：《[智能体名称]》
选手：[姓名]
学校：[学校名称]
```

---

### 段落二：需求分析（30秒）

#### 口播台词

```
[描述背景故事，为什么要做这个智能体]

比如：在日常[学习/生活]中，我发现[某个问题]。
[例如：同学们经常遇到XXX困难/家长担心XXX/同学们对XXX很感兴趣]
但是[现有的XXX]只能[有限的帮助]。

所以我想：能不能用AI做一个[能XXX]的智能体呢？
```

#### 画面内容

- [ ] 展示问题场景的图片或示意图
- [ ] 展示现有解决方案的不足（对比图）

> [镜头提示] 用具体故事引入，让评委有代入感

---

### 段落三：系统演示（约2分钟）

> ⚠️ 这是视频的核心部分，必须展示智能体的实际运行！

#### 演示一：正常流程演示（60秒）

##### 口播台词

```
现在让我来实际演示一下。

[边操作边说，逐步展示]

首先，我[进行什么操作]，比如输入[具体输入内容]。
你可以看到，AI[做了什么处理]，然后[给出了什么结果]。

[展示关键环节]
这里特别要说明的是，AI会根据[什么信息]来做[什么判断]。
如果是[什么情况]，它会走[什么分支]...
```

##### 画面内容

- [ ] **屏幕录制**展示智能体的实际操作过程
- [ ] 标注关键节点（如"意图识别"、"AI判断"、"分支处理"）
- [ ] 突出显示AI的智能决策过程

> [此处插入]操作过程的关键截图

#### 演示二：不同分支/场景演示（40秒）

##### 口播台词

```
[展示另一个使用场景]

当用户[输入不同内容]时，AI会识别到这是[另一种意图]。
然后它会[走另一条处理路径]，给出[不同的响应]。

这体现了我们智能体的[智能决策/多路径处理]能力。
```

##### 画面内容

- [ ] 展示不同分支的对比效果
- [ ] 展示AI判断的变化

#### 演示三：亮点特色（20秒）

##### 口播台词

```
我们智能体还有一个特色功能：[特色功能名称]。

[演示特色功能]

这个功能使得我们的智能体[有什么区别/优势]。
```

##### 画面内容

- [ ] 展示特色功能的操作和效果

---

### 段落四：技术讲解（60秒）

#### 口播台词

```
现在让我简单介绍一下背后的技术设计。

我们使用了Coze平台来搭建这个智能体。

[展示工作流截图]

整个智能体包含[数字]个工作流，共[数字]个处理节点。
核心采用了[设计模式，如：分支式/多智能体协同式]架构。

[指出判断节点]
这里是一个关键的判断节点，它基于大模型的输出来决定走哪个分支。
这是真正的AI智能决策，而不是简单的if-else规则。

[如有Subflow]
我们把[某部分逻辑]拆分成了独立的子流程，这样整个系统更清晰，也更方便调试。

[如有输出校验]
我们还加入了输出校验机制，
用代码节点来验证AI生成的内容是否合格，
不合格会自动重新生成，保证了输出质量。
```

#### 画面内容

- [ ] **展示工作流截图（必须清晰！）**
- [ ] 标注关键节点（LLM节点、判断节点、Subflow调用等）
- [ ] 箭头指引评审关注的重点

> [此处插入截图]工作流整体截图 > [此处插入截图]关键节点细节截图

### 段落五：总结反思（30秒）

#### 口播台词

```
通过这次比赛，我最大的收获是[收获1]和[收获2]。

[具体说明收获，如：学会了用工作流来拆解复杂问题、
理解了AI智能体的决策逻辑、锻炼了项目规划能力等]

未来，我希望把这个智能体[怎么改进和扩展]，
让它能[更大的价值/服务更多的同学]。

谢谢大家！
```

#### 画面内容

- [ ] 展示项目统计（节点数、测试场景数、开发时长等）
- [ ] 展示使用记录（其他同学试用的截图，如有）
- [ ] 致谢页面

## 拍摄Checklist

### 拍摄前准备

- [ ] 智能体已测试通过，各分支运行正常
- [ ] 写好了口播稿，练习过2-3遍
- [ ] 录制环境安静、光线充足
- [ ] 屏幕录制软件已准备好（推荐：OBS/系统自带录屏）
- [ ] 关闭不必要的通知和应用窗口
- [ ] 工作流截图已保存（分辨率≥1280×720）

### 拍摄时注意

- [ ] 全程按脚本顺序录制，不要跳段
- [ ] 操作智能体时语速放慢，让评委看清楚
- [ ] 点击每个按钮前说一下"我现在点击XXX"
- [ ] 等待AI回复时不要沉默，可以简要解说

### 拍摄后处理

- [ ] 剪辑掉明显的口误和停顿
- [ ] 添加必要的字幕说明（作品名、选手名、关键概念）
- [ ] 添加关键节点的标注和箭头
- [ ] 检查视频时长：4分20秒 - 4分50秒为佳
- [ ] 检查文件大小：< 200MB
- [ ] 检查分辨率：≥ 1280×720
- [ ] 检查格式：MP4

## 配音/录制方式选择

| 方式 | 优点 | 缺点 | 推荐场景 |
|------|------|------|----------|
| **直接录制** | 真实自然，有表现力 | 容易紧张口误 | 表达流畅的选手 |
| **先录屏再配音** | 操作展示更稳定 | 容易出现音画不同步 | 操作复杂的演示 |
| **录屏+画中画** | 既看操作也看演讲者 | 需要画中画软件 | **推荐**（最有临场感） |

---

## 紧急应对

| 意外 | 预案 |
|------|------|
| 智能体报错 | 重新演示，并说明"这是测试版本，正式版会更稳定" |
| 忘词 | 看提示卡（放在镜头旁边，自然看过去） |
| 演示太慢 | 可以2倍速播放等待过程，后期加速 |
| 超时 | 精简技术讲解段落，或合并演示一和二 |

# PPT 制作大纲模板

> 本文档供 `/agentmaker` Skill 阶段6使用：生成竞赛报告PPT大纲  
> 结构对齐评审四大板块：需求分析、系统设计、开发与测试、开发反思  
> 建议PPT页数：15-20页

---

## PPT 整体结构

| 板块 | 页码 | 内容 | 建议时间 |
|------|:---:|------|:---:|
| 封面 | 1 | 项目标题 + 选手信息 | — |
| 一、需求分析 | 2-4 | 为什么做/做什么 | 1分钟 |
| 二、系统设计 | 5-9 | 怎么做 | 2分钟 |
| 三、开发与测试 | 10-13 | 做得怎么样 | 1.5分钟 |
| 四、开发反思 | 14-15 | 学到了什么 | 0.5分钟 |
| 结尾 | 16 | 致谢 + 作品链接 | — |

## PPT 设计建议

| 维度 | 建议 |
|------|------|
| **配色** | 不超过3种主色调，保持简洁统一 |
| **字体** | 标题用粗体，正文不小于18号（保证投影可读） |
| **图文比例** | 尽量图文并茂，每页至少有一张图/截图/图表 |
| **字数控制** | 每页要点不超过5条，每条不超过一行 |
| **动效** | 简单淡入即可，不要过度动画 |
| **截图要求** | 工作流截图必须清晰，标注关键节点 |

## 常见问题

| 问题 | 建议 |
|------|------|
| 截图像素太低看不清 | 使用Coze内置的截图功能或系统截图工具，确保分辨率≥1280×720 |
| 不知道怎么画架构图 | 用PPT内置的SmartArt或简单的方框+箭头 |
| 测试结果太多不知道怎么放 | 挑最重要的5-8个测试场景，其余放附录 |
| 内容太多页数超了 | 优先保留需求分析+系统设计+测试结果，开发反思精简 |

# Coze 逐节点配置模板

> 基于Coze平台 2026-05 版本  
> 使用功能名称定位（如"选择「大模型」节点"），不使用具体UI位置描述  
> 每个节点包含：输入变量表、输出变量表、模型配置、完整提示词

---

## 使用说明

1. 在方案设计文档的基础上，为每个流程图节点填写一张以下配置表
2. 学生拿着这份文档，可以直接在Coze中"照表搭建"
3. 所有 `[方括号]` 中的内容是占位符，需要替换为实际内容
4. 截图占位符位置用 `[此处插入截图]` 标记，学生自行截图

### 示例：节点1 — 意图识别

**所属工作流**：`main_workflow`

**节点类型**：大模型

**节点功能**：分析用户输入，识别意图类型

---

#### 输入变量表

| 变量名 | 类型 | 来源 | 说明 |
|--------|------|------|------|
| `user_input` | String | 开始节点的 `USER_INPUT` | 用户原始输入 |
| `session_history` | List(Object) | 开始节点的 `session_history` | 最近对话历史 |

---

#### 输出变量表

| 变量名 | 类型 | 说明 |
|--------|------|------|
| `output` | String | LLM输出的JSON文本，需包含intent、confidence、keywords字段 |

---

#### 模型配置

| 配置项 | 值 |
|--------|-----|
| 模型 | 豆包·1.8·深度思考 |
| responseFormat | JSON |
| 温度 | 0.3 |
| 最大输出Token | 1024 |
| 关联知识库 | 无 |

---

#### 系统提示词

```markdown
你是意图识别模块。请分析用户的输入，判断ta的意图。

## 意图分类规则
- intent_a（意图A）: [意图A的触发条件和关键词]
- intent_b（意图B）: [意图B的触发条件和关键词]
- intent_c（意图C）: [意图C的触发条件和关键词]
- default（默认）: 无法识别时的兜底分类

## 输出格式（严格按JSON格式输出）
{
  "intent": "intent_a/intent_b/intent_c/default",
  "confidence": 0.95,
  "keywords": ["关键词1", "关键词2"]
}

## 注意事项
1. confidence是0-1之间的浮点数，表示置信度
2. 如果用户输入模糊，优先选择最可能的意图
3. 仅输出JSON，不要添加任何解释文字
```

## 七、配置对照检查清单

搭建完成后对照以下清单逐项检查：

### 节点连接检查
- [ ] 所有节点的输入输出都已正确连接
- [ ] 各分支最终都汇聚到变量聚合或结束节点
- [ ] 没有"悬空"的节点（有输入没有输出）

### 变量引用检查
- [ ] 所有 `{{变量名}}` 引用路径正确
- [ ] 子流程的输入参数与子流程开始节点定义一致
- [ ] 变量类型匹配（String传给String，List传给List）

### 选择器条件检查
- [ ] 条件表达式中的变量路径正确
- [ ] 每个选择器都有ELSE默认分支（兜底）
- [ ] 条件分支的覆盖是完备的（不会出现"无分支匹配"的情况）

### 提示词检查
- [ ] 每个LLM节点有独立的、差异化的系统提示词
- [ ] 提示词中使用的变量名称与输入变量表一致
- [ ] JSON输出的LLM节点已设置 `responseFormat: JSON`
- [ ] 提示词中不含过时的UI描述

### 输出校验检查
- [ ] 至少1个代码节点做LLM输出校验
- [ ] 校验失败后有重试或兜底处理
- [ ] 校验内容覆盖了格式、内容安全性、必填字段

### 节点数量检查
- [ ] 节点数达到组别要求（小学≥5，初中/高中≥10）
- [ ] 包含至少1个基于LLM输出的判断节点

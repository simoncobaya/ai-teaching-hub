# Skill 导出模板

> 本文档用于 `/agentmaker` 阶段3：将工作流设计映射为 CodeBuddy Skill  
> 填空式模板：将每个 workflow 节点映射为 SKILL.md 执行步骤

---

## 一、YAML Front Matter 模板

```yaml
---
name: [skill_name]
description: >
  [技能描述，2-3句话说明这个Skill做什么]
  主要功能：
  1. [功能1]
  2. [功能2]
  适用场景：[场景描述]
---

# /[skill_name] — [中文名称] Skill
```

---

## 二、SKILL.md 正文模板

### 触发方式

```markdown
## 触发方式

通过以下任一方式触发本 Skill：

- 发送命令：`/[skill_name]`
- 发送关键词：`[关键词1]`、`[关键词2]`
- 场景触发：[触发场景描述]
```

---

### 使用方式

```markdown
## 使用方式

使用格式：

\`\`\`
/[skill_name] [用户输入内容]
\`\`\`

示例：

\`\`\`
/[skill_name] [示例输入1]
/[skill_name] [示例输入2]
\`\`\`
```

---

### 执行流程

> 每一个 Workflow 节点映射为以下的一个步骤（H3标题）
> Subflow 映射为子步骤（加粗列表项 + 缩进）

```markdown
## 执行流程

### 第1步：[节点1名称 - 对应LLM/处理节点]

[节点功能描述]

- **输入**：[说明接受什么输入]
- **处理**：[说明做什么处理]
- **输出**：[说明输出什么结果]

### 第2步：[节点2名称 - 对应选择器/判断节点]

根据第1步的[输出字段]进行判断：

- **如果[条件A]** → 执行第3-A步
- **如果[条件B]** → 执行第3-B步
- **其他情况** → 执行兜底处理

### 第3-A步：[分支A处理节点]

[处理描述]

- 调用 [Subflow名称，对应subflow节点]：
  - [子步骤1]
  - [子步骤2]
  - [子步骤3：输出校验 - 对应代码校验节点]

### 第3-B步：[分支B处理节点]

[处理描述]

### 第N步：输出汇总

将各分支的结果汇总为最终输出。

### 第N+1步：输出校验（通用步骤）

在输出最终结果前，进行以下校验：

1. **[校验规则1]**：[校验内容]
2. **[校验规则2]**：[校验内容]
3. **[校验规则3]**：[兜底处理]

### 第N+2步：生成[输出卡片/报告名称]

\`\`\`
[输出格式模板]
\`\`\`
```

---

## 三、Subflow 结构 → SKILL.md 步骤映射规则

| Workflow/Subflow 元素 | SKILL.md 对应内容 |
|------------------------|-------------------|
| 开始节点（输入变量） | 步骤开头的「输入」说明 |
| LLM节点 | 独立的H3步骤标题 |
| Condition选择器 | 步骤中的分支判断（"如果...则..."列表） |
| Subflow调用 | 步骤中的子步骤（加粗 + 缩进） |
| 代码校验节点 | 「输出校验」步骤中的校验规则 |
| 变量聚合节点 | 「输出汇总」步骤 |
| 结束节点（输出变量） | 最后一步的输出格式 |
| Loop循环 | 步骤中的「重试机制」说明 |

## 四、常用执行步骤模板

### 模板A：意图识别 + 路由选择

```markdown
### 第1步：意图识别

分析用户输入，判断用户的意图类型。

- **输入**：用户输入的文本
- **处理**：使用大模型分析意图，分类为 [A/B/C/D]
- **输出**：意图类别 + 置信度 + 关键词

### 第2步：意图路由

根据意图识别结果进行路由：

- **如果意图 = [A]** → 执行第3-A步: [功能描述]
- **如果意图 = [B]** → 执行第3-B步: [功能描述]
- **如果意图 = [C]** → 执行第3-C步: [功能描述]
- **其他情况** → 执行兜底处理
```

### 模板B：LLM分析 + 条件判断

```markdown
### 第X步：[分析名称]

使用大模型对 [分析对象] 进行 [分析维度]。

- **输入**：[输入内容]
- **角色设定**：你是一个 [角色描述]，负责 [分析任务]
- **分析维度**：[维度1]、[维度2]、[维度3]
- **输出**：[结构化输出格式]

### 第X+1步：[判断名称]

根据 [分析] 的结果：

- **如果[结果=A]**：[处理方式A]
- **如果[结果=B]**：[处理方式B]
```

### 模板C：知识库检索（Subflow）

```markdown
### 第X步：知识检索

根据 [检索关键词] 在知识库中检索相关内容。

- 检索范围：[知识库名称]
- 检索策略：混合检索（语义 + 关键词）
- 返回 Top-K 结果：[数量]
- 如无匹配结果：[兜底处理]
```

### 模板D：输出校验（Code节点 → 重试/兜底）

```markdown
### 第Y步：输出校验

对第X步的输出进行校验：

1. **格式校验**：检查 [格式要求]
2. **内容校验**：检查 [内容要求]
3. **安全校验**：过滤 [敏感内容类型]

- **校验通过** → 使用该结果
- **校验不通过，重试 < N次** → 回到第X步重新生成
- **重试超过N次** → 使用兜底内容：[兜底文案]
```

### 模板E：循环交互（Loop/Question → 条件回路）

```markdown
### 第X步：[交互轮次名称]

进行第 [N] 轮 [交互类型]：

- **提问**：向用户展示 [问题内容]
- **等待输入**：收集用户的回答
- **分析**：对回答进行评估

**循环判断**：
- **如果[继续条件]** → 回到本步骤，进行下一轮
- **如果[结束条件]** → 进入下一步汇总
```

## 七、映射示例：从工作流到SKILL.md

### 示例工作流（简化版交易风险判断）

```
开始 → price_analysis(LLM) → tactics_analysis(LLM) 
     → method_analysis(LLM) → risk_assessment(LLM) 
     → 选择器(高/中/低) → 分路输出 → 变量聚合 → 结束
```

### 映射后的SKILL.md步骤

```markdown
## 执行流程

### 第1步：价格分析
分析商品价格的合理性。

- **输入**：商品描述和标价
- **处理**：对比市场价，判断价格是否异常
- **输出**：价格合理性评估

### 第2步：话术分析
分析卖家的描述话术是否有风险特征。

- **输入**：商品描述文本
- **处理**：检测催促、恐吓、虚假承诺等风险话术
- **输出**：话术风险评估

### 第3步：交易方式分析
分析建议的交易方式是否安全。

- **输入**：交易方式描述
- **处理**：判断是否涉及私下转账、陌生链接等
- **输出**：交易方式安全评估

### 第4步：综合风险评估
综合三个维度的分析结果，给出最终风险等级。

- **输入**：价格分析 + 话术分析 + 交易方式分析的结果
- **处理**：加权综合三个维度的风险
- **输出**：风险等级（高/中/低）+ 风险说明

### 第5步：风险分级输出

- **如果风险=高** → 强烈建议不要交易并说明原因
- **如果风险=中** → 列出需要注意的风险点
- **如果风险=低** → 确认安全并给出正常交易建议

### 第6步：生成风险评估报告

\`\`\`
🔍 交易安全评估报告
━━━━━━━━━━━━━━━
📊 风险等级：[高/中/低]
💰 价格：[合理/偏高/异常低]
🗣️ 话术：[正常/有疑虑/高危]
💳 交易方式：[安全/需注意/危险]

📝 详细分析：
[分析内容]

⚠️ 建议：
[建议内容]
\`\`\`
```

## 八、Skill 阶段与 Coze 阶段的对应

| Skill 步骤 | 对应 Coze 节点 | 说明 |
|------------|---------------|------|
| 第1步：意图识别 | LLM节点（意图分类） | Skill中为自然语言描述，Coze中为精确提示词 |
| 第X步：条件路由 | Condition选择器 | Skill中用if-else文字，Coze中为条件表达式 |
| 子步骤（缩进） | Subflow调用 | Skill中展开描述，Coze中为独立workflow |
| 输出校验步骤 | Code节点 | Skill中为规则描述，Coze中为JS/Python代码 |
| 输出汇总步骤 | Variable Merge节点 | Skill中为自然语言汇总 |

---

> 📝 **使用建议**：在阶段2完成方案设计后，将工作流图中的每个节点填入本模板，生成 SKILL.md 初稿。在 CodeBuddy 中测试，逻辑确认无误后再去 Coze 搭建。
